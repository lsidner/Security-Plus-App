7. Sarah is a cybersecurity analyst at a growing e‐commerce company. She’s tasked with enhancing the company’s email security measures. D. SPF, DKIM, and DMARC

Answer: D. SPF, DKIM, and DMARC
Explanation: To enhance email security measures, Sarah should deploy SPF (Sender Policy Framework), DKIM (DomainKeys Identified Mail), and DMARC (Domain-based Message Authentication, Reporting, and Conformance). Deploying all three protocols will provide comprehensive email security for the company.

28. What is the key difference between checksums and hashes?

A. Checksums are unique to every file; hashes are not.

B. Hashes are reversible and can be used to generate the original file or data; checksums cannot be used to re‐create the original file.

C. Checksums are a small value used to detect errors, whereas hashes map to an arbitrarily sized output that is unique to each file hashed.

D. There is no difference—checksums and hashes are the same.

Answer: C. Checksums are a small value used to detect errors, whereas hashes map to an arbitrarily sized output that is unique to each file hashed.
Explanation: Checksums and hashes are both used to verify the integrity of data, but they serve different purposes. A checksum is a small value calculated from a data set that is used to detect errors in the data. It is typically a simple mathematical function that produces a fixed-size output. On the other hand, a hash is a more complex function that maps input data to an arbitrarily sized output, which is unique to each file or data set hashed. Hashes are designed to be irreversible, meaning that it should not be possible to generate the original file or data from the hash value. This makes hashes more suitable for security applications, while checksums are primarily used for error detection.

30. Simon is a system administrator for a technology firm. His organization has high data modification and transaction rates. A. RAID 0

B. RAID 1

C. RAID 5

D. RAID 10

Answer: C. RAID 5
Explanation: RAID 5 is a suitable choice for Simon's organization because it offers high read speeds and fault tolerance for at least one disk failure while not requiring double the storage. RAID 5 uses striping with parity, which allows it to distribute data across multiple disks while also storing parity information that can be used to recover data in the event of a single disk failure. Therefore, RAID 5 is the best option for Simon's organization.

D. $75,000

B. Input allow lists

C. Browser‐based input validation

D. Signature detection

Answer: B. Input allow lists
Explanation: Input allow lists (also known as whitelisting) are the most effective way to defeat injection attacks because they specify exactly what input is considered valid and reject anything that does not conform to those specifications. Therefore, the correct answer is input allow lists.
